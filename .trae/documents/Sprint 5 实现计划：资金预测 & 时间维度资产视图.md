## Sprint 5 实现计划

### 核心目标
从时间维度回答：未来一段时间内，我的资产流动性结构可能会如何变化？

### 功能模块

#### S5-1: 时间轴资产视图（核心）
- **API**: `GET /api/dashboard/cash_timeline`
- **展示**: Today / +7d / +30d / +90d 的流动性结构
- **包含**: 当前可用现金、在途赎回、锁定资金

#### S5-2: 未来现金区间汇总
- **扩展**: Dashboard 增加 90 天区间
- **API**: 扩展 `future_cash_flow` 支持 90 天
- **展示**: 7/30/90 天预计可用金额

#### S5-3: 产品层面时间提示（可选）
- **API**: `GET /api/products/{id}/liquidity_status`
- **展示**: 锁定期状态、最近变现时间
- **位置**: 产品详情页

### 实现步骤

1. 后端：扩展 cash_service，增加时间轴计算
2. 后端：新增 cash_timeline API
3. 后端：新增 product liquidity_status API
4. 后端：扩展 future_cash_flow 支持 90 天
5. 前端：新增 Dashboard 时间轴组件
6. 前端：扩展 Dashboard 展示 90 天预测
7. 前端：产品详情页增加流动性提示
8. 更新 OpenAPI 规范

### 技术要点
- 复用 Sprint 4 数据逻辑（在途赎回、未来现金流）
- 明确标注"预计/规则推算"，不做精度幻觉
- 不引入 Lot/批次模型
- 不做收益预测

### 验收标准
- [ ] Dashboard 显示未来 7/30/90 天预计可用现金
- [ ] 时间轴清晰展示资金流动性变化
- [ ] 产品详情页显示锁定期状态
- [ ] 所有预测明确标注"预计/规则推算"
